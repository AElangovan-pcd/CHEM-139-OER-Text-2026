# Variable-Problem Pilot — Phase 3b Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship 8 new variable problems (2 mass-percent in Ch 8 §8.4, 6 stoichiometry in a new Ch 9 spec) on top of one new engine operation (`mass_percent` with `decimal_places` precision) and one new variable generator (`pick_one` with struct unfolding).

**Architecture:** Layer minimal additions on top of the existing engine — Phase 3a's `factor_label` carries the 6 stoichiometry problems unchanged; the 2 mass-percent problems exercise the new `mass_percent` operation. The `pick_one` generator returns option structs which `generateVariant` unfolds into prefixed flat keys, so every downstream consumer (`substituteTemplate`, `renderLatexForOperation`, etc.) keeps seeing flat string-valued params.

**Tech Stack:** ES module JavaScript (Node 20+ for tests, browser ES module at runtime), Python 3.11 (mammoth + python-docx + pyyaml + beautifulsoup4 + unittest), MathJax v3 (browser-only), GitHub Pages CI.

**Spec:** `docs/superpowers/specs/2026-05-08-variable-problems-phase3b-design.md` (committed on branch `pilot-variable-problems-phase3b` at commits `0551de7` + `7ba1583`).

**Branch:** `pilot-variable-problems-phase3b` (already exists, off `main` at `a37815a`).

**Out of scope:** Textbook prose insertions in Ch 1 §1.12, Ch 8 §8.3, Ch 9 §9.7. Those are docx edits applied separately by the author in Word per the spec's §3.1.

```javascript
/**
 * Mass-percent operation: (partial / total) * 100, rounded to N decimal places.
 * Sister to to_sci_notation in that the precision idiom (decimals, not sig figs)
 * is operation-specific. Author chooses decimal_places per the textbook
 * convention (2 for light-element compounds, 1 for any-heavy compound).
 */
export function computeMassPercent(answerSpec, params) {
  const partial = parseFloat(params[answerSpec.partial_mass_param]);
  const total = parseFloat(params[answerSpec.total_mass_param]);
  const decimals = answerSpec.decimal_places ?? 2;
  const rawPercent = (partial / total) * 100;
  const finalPercent = rawPercent.toFixed(decimals);
  return {
    rawPercent: rawPercent.toString(),
    finalPercent,
    finalPercentLatex: finalPercent,
  };
}
```

```javascript
export function generateVariant(problemSpec, rng) {
  for (let i = 0; i < MAX_RETRIES; i++) {
    const params = {};
    for (const [name, varSpec] of Object.entries(problemSpec.variables || {})) {
      const value = sampleValue(varSpec, rng);
      if (value !== null && typeof value === 'object') {
        // pick_one returned a struct → unfold into prefixed flat keys.
        for (const [k, v] of Object.entries(value)) {
          params[name + '_' + k] = String(v);
        }
      } else {
        params[name] = value;
      }
    }
    const computed = computeAnswer(problemSpec.answer, params);
    if (passesGuardrails(problemSpec.constraints || {}, params, computed)) {
      return { params, computed };
    }
  }
  throw new Error('Guardrail cap hit (' + MAX_RETRIES + ' retries) for spec: ' + problemSpec.id);
}
```
